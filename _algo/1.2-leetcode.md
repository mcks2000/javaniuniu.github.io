# 审题
- 题目起码要理解透
- 可用算法及思路(思路一闪而过，可以记录下来)
- 画出判断条件
- 编码

# 方法
- 在编辑代码的过程中，思路会越渐清晰(通常，会在有一点思路的时候会编辑伪代码)
- 暴力解法 可以先做一遍，可以缕清很多思路
- 二叉树配合往往递归 对于递归需要判断
- 用递归或者迭代代替 for while 循环，可以大大降低时间复杂度
- 对数组中元素没有顺序要求题目，可对数组排序
- 当暴力破解超时，可以思考是不是可以降低纬度
- 判断循环就用快慢指针，在链表中常出现
